# 版本检查接口文档

## 接口地址

App 通过静态 JSON 文件获取版本信息（无后端依赖）：

```
https://kotier.openstars.org/version.json
```

## JSON 格式

```json
{
  "latest_version": "4.2.45",
  "download_url": "https://kotier.openstars.org/downloads/kotier-v4.2.45-4063-release.apk",
  "release_notes": "4.2.45 发版"
}
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| latest_version | string | 是 | 最新版本号，如 "4.2.45"（不要带 v 前缀） |
| download_url | string | 是 | APK 下载地址 |
| release_notes | string | 否 | 更新说明 |

## 数据同步方式

服务器使用自动同步脚本轮询 GitHub Releases：

1. **服务器**每天北京时间 02:00 自动检查 GitHub Releases（cron）
2. 发现新版本后自动下载 APK 到服务器 `downloads/` 目录
3. 自动更新 `version.json` 中的版本号和下载地址
4. App 直接读取静态 JSON，无需后端 API

同步脚本位置：`scripts/sync-apk.js`

## 更新版本流程（管理员）

1. 在 GitHub 上发布新 Release（push tag 触发 Actions 打包）
2. 服务器每日凌晨自动同步，无需手动操作
3. 如需立即更新，可登录服务器手动执行：
   ```bash
   cd /path/to/kotier.openstars
   node scripts/sync-apk.js
   ```

## 注意事项

- 确保服务器支持 HTTPS（现代 Android 要求）
- APK 文件存放在 `downloads/` 目录，与静态站点一起部署
- 版本号格式建议使用语义化版本：`主版本.次版本.修订号`

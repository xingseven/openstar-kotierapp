# 版本检查接口文档

## 接口地址

在你的网站上创建一个 JSON 文件，例如：

```
https://your-domain.com/api/version.json
```

## JSON 格式

```json
{
  "latest_version": "1.0.0",
  "download_url": "https://your-domain.com/downloads/app-release.apk",
  "release_notes": "修复了一些bug\n新增了深色模式\n优化了性能"
}
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| latest_version | string | 是 | 最新版本号，如 "1.0.0"（不要带 v 前缀） |
| download_url | string | 是 | APK 下载地址 |
| release_notes | string | 否 | 更新说明，支持换行 `\n` |

## 示例

```json
{
  "latest_version": "1.2.5",
  "download_url": "https://kotier.openstars.org/downloads/app-release.apk",
  "release_notes": "V1.2.5 更新内容：\n- 修复状态栏在深色模式下图标不显示的问题\n- 屏蔽 183 节点\n- 优化检查更新功能，国内用户可正常使用"
}
```

## 网站部署方式

### 方式一：静态 JSON 文件

最简单的方式，直接上传 JSON 文件到你的网站目录：

```
/public_html/api/version.json
```

### 方式二：API 接口

如果你使用后端语言（PHP/Node.js/Python 等），可以动态生成：

**PHP 示例：**

```php
<?php
header('Content-Type: application/json');

$version = [
    'latest_version' => '1.2.5',
    'download_url' => 'https://your-domain.com/downloads/app-release.apk',
    'release_notes' => "V1.2.5 更新内容：\n- 修复状态栏问题\n- 屏蔽 183 节点"
];

echo json_encode($version, JSON_UNESCAPED_UNICODE);
?>
```

**Node.js/Express 示例：**

```javascript
const express = require('express');
const app = express();

app.get('/api/version.json', (req, res) => {
    res.json({
        latest_version: '1.2.5',
        download_url: 'https://your-domain.com/downloads/app-release.apk',
        release_notes: 'V1.2.5 更新内容：\n- 修复状态栏问题\n- 屏蔽 183 节点'
    });
});

app.listen(3000);
```

## 更新版本流程

1. 构建新版本 APK
2. 上传到你的网站下载目录
3. 修改 `version.json` 中的版本号和下载链接
4. 完成

## 注意事项

- 确保你的网站支持 HTTPS（现代 Android 要求）
- 下载链接建议使用直链，避免跳转
- 版本号格式建议使用语义化版本：`主版本.次版本.修订号`
